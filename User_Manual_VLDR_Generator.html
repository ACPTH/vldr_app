VLDR Generator — User Manual

Version: 1.0 • Language: English

1. Overview

The VLDR Generator creates Vehicle Loss & Damage Report PDFs from Excel files. It supports multiple OEM templates (BMW, FCA, Renault, Stellantis, etc.) and allows batch downloads.

2. Access & Login

  1. Open the app URL in your browser.
  2. Log in with your username and password.
  3. If you have admin privileges, an Admin link will appear in the top bar.
Screenshot placeholder: Login screen

3. Uploading an Excel File

  1. Drag & drop an Excel file into the drop zone, or click to upload.
  2. The app reads the sheet named damage_list.
  3. Once loaded, the VIN table appears.
Required columns (typical): vin, make, model, date, damage_part_code, damage_type_code, damage_extent_code, damage_classification, damage_remark.

4. Understanding the Table

  • VIN: Unique vehicle identifier.
  • Format: Auto-selected by make (can be overridden).
  • Damage columns: Part, Type, Extent, Classification.
  • Remark: You can edit; the value is applied to all damage lines for that VIN.
Screenshot placeholder: VIN table

5. Format Selection

Click the format pill to choose a different template. If your user account is restricted, you will only see allowed formats.

Screenshot placeholder: Format dropdown

6. Manual Fields

Some formats require extra fields (e.g., Vessel, Customer, Transport Ref). When you download, a modal appears to fill these values.

Screenshot placeholder: Manual fields modal

7. Preview

Click Preview to generate a single-PDF preview for one VIN.

8. Download Options

  • Download Selected (ZIP): One PDF per VIN.
  • Download All (ZIP): One PDF per VIN for all rows.
  • Download All Merged: One merged PDF per format.

9. Queue & Load Status

The status bar displays active jobs, queued jobs, and average time. If the server is busy, requests will wait briefly instead of failing immediately.

Screenshot placeholder: Queue/Load status bar

10. Admin Panel

Admins can create users, reset passwords, enable/disable accounts, and limit formats per user.

  • Open /admin in the browser.
  • Use the Formats checkboxes to restrict each user.
Screenshot placeholder: Admin panel

11. Troubleshooting

  • “Server not responding”: Check if the backend is running or redeployed.
  • Blank PDFs: Ensure templates are correct and pdftk is installed on the server.
  • Login fails: Verify DATABASE_URL and Supabase credentials.

12. Tips for Faster Output

  • Limit concurrency with VLDR_MAX_JOBS.
  • Use a paid Render plan for more CPU/RAM.
  • Keep templates optimized and avoid oversized fields.
